# 🔍 Как проверять изменения перед загрузкой на GitHub

Эта инструкция для тебя — чтобы ты мог сам проверять изменения перед тем, как заливать их в репозиторий.

## 📁 Структура проекта

```
therapy-funnel/
├── index.html          # Главная страница (весь тест)
├── check-changes.sh    # Скрипт проверки изменений
├── preview.sh          # Скрипт локального предпросмотра
├── deploy.sh           # Скрипт деплоя на GitHub
├── upload-to-github.sh # Скрипт загрузки с токеном
└── ... остальные файлы
```

## 🔄 Процесс работы (шаг за шагом)

### 1. Внес изменения в код
```bash
# Редактировал index.html или другой файл
# Например, добавил новые варианты ответов
```

### 2. Проверить изменения локально
```bash
# Вариант А: Быстрый просмотр
bash preview.sh
# Откроет браузер на http://localhost:3000
# Пройди тест, убедись что всё работает

# Вариант Б: Только открыть в браузере
open index.html
```

### 3. Увидеть что именно изменилось
```bash
bash check-changes.sh
```
Скрипт покажет:
- Какие файлы изменились
- Конкретные изменения (diff)
- Предложит создать коммит

### 4. Создать коммит (если всё ок)
```bash
# Вручную:
git add .
git commit -m "Описание изменений"

# Или через скрипт (предложит сам):
bash check-changes.sh
# → ответить 'y' на вопрос о коммите
```

### 5. Отправить на GitHub
```bash
git push
```

## 🛠️ Скрипты-помощники

### `preview.sh` — предпросмотр
```bash
bash preview.sh
```
Что делает:
1. Запускает локальный веб-сервер
2. Открывает браузер с тестом
3. Показывает адрес (http://localhost:3000)

### `check-changes.sh` — проверка изменений
```bash
bash check-changes.sh
```
Что делает:
1. Показывает `git status`
2. Показывает diff всех изменённых файлов
3. Предлагает создать коммит
4. Спрашивает сообщение коммита
5. Предлагает отправить на GitHub

## 📝 Работа с Git (если хочешь вручную)

### Посмотреть изменения
```bash
# Какие файлы изменились
git status

# Конкретные изменения в файле
git diff index.html

# Изменения в виде списка
git diff --stat
```

### Отменить изменения (если что-то пошло не так)
```bash
# Отменить изменения в конкретном файле
git checkout -- index.html

# Убрать файл из staged (если добавил по ошибке)
git reset HEAD index.html

# Отменить последний коммит (но оставить изменения)
git reset --soft HEAD~1
```

### Работа с коммитами
```bash
# Добавить все изменения
git add .

# Добавить конкретный файл
git add index.html

# Создать коммит
git commit -m "Добавил пятый вариант ответа"

# Отправить на GitHub
git push

# Посмотреть историю коммитов
git log --oneline
```

## ⚠️ Частые ситуации

### «Я сделал изменения, но не вижу их в браузере»
1. Убедись что сохранил файл (Ctrl+S или Cmd+S)
2. Перезагрузи страницу (Cmd+R или F5)
3. Если запущен `preview.sh` — сервер обновляет автоматически

### «Хочу посмотреть как было до изменений»
```bash
# Показать изменения с последнего коммита
git diff HEAD

# Показать изменения конкретного файла
git diff HEAD index.html
```

### «Случайно залил не то»
```bash
# Отменить последний пуш (осторожно!)
git reset --hard HEAD~1
git push --force

# Или создать новый коммит с исправлениями
# (лучше и безопаснее)
```

## 🚀 Быстрая команда для проверки всего

Создай алиас в терминале:
```bash
# Добавь в ~/.zshrc или ~/.bashrc
alias check-therapy='cd ~/Downloads/therapy-funnel && bash check-changes.sh'
```

Потом просто:
```bash
check-therapy
```

## 📞 Если что-то не работает

1. **Скрипты не запускаются:**
   ```bash
   chmod +x *.sh
   ```

2. **Git не видит изменения:**
   ```bash
   git status
   # Если файлы не отслеживаются:
   git add .
   ```

3. **Локальный сервер не запускается:**
   ```bash
   # Установи npx если нет
   npm install -g npx
   # Или используй python
   python3 -m http.server 8000
   ```

## 🎯 Главное правило

**Сначала проверяй → потом коммитишь → потом пушишь.**

Так всегда будешь знать, что на GitHub попадает только проверенный код.
